| URL Shortener Example (contributor) | #239 | Adds `examples/url_shortener/` — a CRUD `link` domain + a Taskiq `cleanup_expired_links_task` that shares the same `LinkService` as the HTTP router (mirrors `examples/todo/` layout + the `src/user` worker pattern). Endpoints `POST /v1/link`, `GET`/`DELETE /v1/link/{short_code}` (path param bounded `Path(min_length=1, max_length=64)`). README documents the InMemory single-process enqueue recipe: the snippet must bootstrap `UrlShortenerContainer` on the shared `CoreContainer` + `wire()` the task module before `.kiq()`, since the InMemory broker runs the task inline in the calling process. | Core not-found 404 fix | #246 | `Database.session()` caught every in-block exception — including a domain `BaseCustomException` (e.g. the 404 a repository raises on a missing row) — and re-wrapped it as `DatabaseException(500, DB_INTERNAL_ERROR)`, so every domain's "get by id → missing row" returned 500 instead of 404 (and in dev leaked the internal message into `errorDetails.original_error`). Adds `except BaseCustomException: raise` ahead of the catch-all so typed domain/infra exceptions (and already-typed `DatabaseException`) propagate unchanged; only genuine driver errors become a 500. Tightened the docs e2e assertion `(404, 500)` → `404` + added `test_database_session.py` regression coverage. |
| Simple Chatbot Example (contributor) | #249 | Adds `examples/simple_chatbot/` — the project's **first example that calls a real external LLM**. Mirrors `src/classification`'s Protocol + Adapter + Selector + Stub wiring → degrades to a deterministic `StubChatbot` when no LLM provider is configured. Deliberately minimal (no endpoint auth; runtime guardrails deferred to #250; `tokens_used` not routed through `ai_usage` #75) with production caveats documented in the example README. `examples/`-only — no `src/` runtime change. Shipped in v0.8.2 alongside the examples-index production-surface distinction (#253). Closes #97. |
| Chatbot Examples: Memory + Guardrails (contributor) | #255 / #256 | Two new LLM-calling `examples/`. `chatbot_with_memory/` (#255, closes #251) persists each turn and replays session history into a PydanticAI `Agent` via structured `message_history`, so the model sees the conversation so far. `chatbot_with_guardrails/` (#256, closes #250) wraps a PydanticAI `Agent` with the shared `src/_core/infrastructure/llm/guardrails.py` — a prompt-injection input guard blocks with `400 PROMPT_INJECTION_DETECTED`, reusing the existing guardrail exceptions + telemetry with a `guardrails_enabled` kill-switch honored on both the real and stub paths. Both mirror the `src/classification` Protocol + Adapter + Selector + Stub wiring and degrade to a deterministic stub with no LLM provider. `examples/`-only — no `src/` runtime change. Shipped in v0.8.3. |
| Examples copy-into-`src/` fix | #262 | Single-domain examples referenced their own containers via absolute `from examples.<name>...` imports + string-path `wire()`, so `cp -r examples/<name> src/<name>` left each router's `Provide[...]` markers on the `examples.*` classes while auto-discovery instantiated the `src.*` ones → every copied example returned `500` at request time. Intra-example imports are now package-relative and bootstraps `wire(modules=[...])` the imported module object; `todo`/`url_shortener`/`webhook_receiver`/`simple_chatbot`/`chatbot_with_memory`/`chatbot_with_guardrails` all boot + serve after copy-in (verified via `make quickstart`). The two-domain `blog` example is a separate cross-domain case, tracked as a follow-up (#261). Shipped in v0.8.3. |
| Blog Example Copy-Flow Fix | #261 (PR #263) | The two-domain `blog` example crashed on the documented cp-to-`src/` activation: `post_container` pulled `examples.blog.author.AuthorModel` while auto-discovery had already registered `src.author.AuthorModel` for the same table (`Table 'author' is already defined`). Intra-domain imports are now package-relative + bootstraps `wire(modules=[...])` (the #262 pattern); the two cross-domain references (post_service → author protocol, post_container → `AuthorRepository`) become runtime-absolute `src.author.*` — the same shape as the real `src/auth → src/user` dependency, commented as unresolvable until the copy. `tests/unit/blog/conftest.py` builds the copied layout at collection time (temp copy + `src.__path__` extension) so unit tests import the exact modules the copied app runs. Verified end-to-end: both domains copied → quickstart boots → `authorDisplayName` resolves. Blog README gains an "Import Layout (copy-flow)" section + camelCase response-key fix. |
