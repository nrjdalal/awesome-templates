ZeroStarter ships `.vscode/` config so your editor lints and formats with the same [Oxc](https://oxc.rs) toolchain the CLI and git hooks use: one Rust tool in place of ESLint and Prettier, fast enough to run on every save.

## VS Code

Open the repo and VS Code recommends one extension, from `.vscode/extensions.json`:

```json
{
  "recommendations": ["oxc.oxc-vscode"]
}
```

Install the [Oxc extension](https://marketplace.visualstudio.com/items?itemName=oxc.oxc-vscode) and it lints (Oxlint) and formats (Oxfmt) inline. `.vscode/settings.json` makes it the default formatter and runs it on save:

```json
{
  "editor.defaultFormatter": "oxc.oxc-vscode",
  "editor.formatOnSave": true
}
```

The extension reads the project's root config, so the editor and the CLI can never disagree:

- `.oxlintrc.jsonc`: lint rules.
- `.oxfmtrc.jsonc`: formatting: no semicolons, sorted imports, Tailwind class sorting. It ignores `.agents/**`, so the AI skill docs are left untouched.

## Cursor

[Cursor](https://cursor.sh) reads the same `.vscode/` config, so the extension and format-on-save work with no extra setup.

## From the terminal

No editor integration required. The same checks run from the shell, and again on commit:

```bash
bun run format        # format the tree with Oxfmt
bun run format:check  # check formatting without writing
bun run lint          # lint the tree with Oxlint
```
