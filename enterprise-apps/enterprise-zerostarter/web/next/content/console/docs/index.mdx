This documentation is private. It is served only inside the access-protected
`/console` area and is never exposed through public search, the sitemap, or the
`llms.txt` indexes.

## Access

Console access requires `user.role === "admin"` ([Better Auth Admin plugin](https://www.better-auth.com/docs/plugins/admin)). There is no env list; the `role` is set explicitly.

- **Bootstrap the first admin:** `bun run console:roles grant you@example.com` (also `revoke <email>` and `list`), or `bun run db:studio` / `UPDATE "user" SET role = 'admin' WHERE email = '…';`.
- **Manage the rest from the console:** an admin sets others' `user.role` via the plugin's `setRole` (the console user-management dashboard). Roles are `admin` and the default `user`; finer tiers can be added later via the plugin's access control.
- The guard reads the session with `disableCookieCache`, so a grant/revoke takes effect on the next `/console` request (it does not wait out the 5-minute `session.cookieCache`). To cut an account off everywhere immediately, ban it or revoke its sessions.

## Adding pages

Drop MDX files into `web/next/content/console/docs`. They are loaded through the
dedicated `consoleDocs` Fumadocs collection and inherit the same access checks
as the rest of the console.
